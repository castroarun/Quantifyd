Pipeline · Pre-Market Brief

A morning intelligence brief, written by a cloud agent, sent from a VPS at 08:03 IST.

Each weekday at 08:00 IST, a Python service on the trading VPS pulls live market data, holdings calendar events, and overnight news. A Claude Code routine in the cloud reads it, writes the narrative, and posts back. The VPS sends the email. The whole loop closes in three minutes.

01 · Architecture

Two halves, one loop.

The VPS handles deterministic work — fetching, persisting, sending. The cloud routine handles the part that benefits from language understanding — sentiment, narrative, judgment. Each side stays in its lane.

VPS
Contabo · Python service
Runs APScheduler inside Gunicorn. Pulls structured data from yfinance, NSE, BSE, and the local holdings_events.db. Persists raw JSON. Owns the SMTP transport because Gmail MCP only exposes create_draft, not send.
yfinance SQLite Gmail SMTP
CLD
Claude Code · cloud routine
Spawned daily at 08:02 IST as a fresh sandboxed session. curls the VPS for raw data, applies sentiment tags to the day's headlines, drafts the narrative One-Liner, and POSTs the synthesis back. Stateless — no secrets, no SSH key, no persistence.
Claude Cloud routine Stateless
02 · The 8-minute window

From wake-up to inbox.

Four scheduled events. The fallback at 08:08 is the safety net — the operator always gets a brief, even if the cloud routine fails.

08:00 IST
VPS builds the raw brief
Job · premarket_brief_build · APScheduler
Parallel fetch from 14 yfinance tickers (GIFT Nifty, S&P 500, US VIX, USDINR, India VIX, Brent, Gold, 10Y UST, DXY, Nikkei, Hang Seng, Kospi). RSS pulls from Moneycontrol, Mint, Reuters India. Reads holdings_events.db for today's results / ex-div / splits on holdings. Downloads the F&O ban-list CSV from NSE archive. Computes a rule-based bias verdict as a fallback. Writes the JSON to disk.
14 market quotes 3 RSS sources F&O ban CSV Holdings DB
08:02 IST
Cloud routine wakes up
Claude Code · scheduled routine
curl http://<vps>:5000/api/premarket/brief/raw. Receives the structured dict. Applies sentiment tags (POS / NEU / NEG) to the top 5 headlines. Drafts a 2-sentence narrative One-Liner that ties GIFT premium, S&P direction, and the day's volatility flags into a single read. POSTs the synthesis back as JSON.
Sentiment tagging Narrative synthesis Holdings-aware
08:03 IST
VPS dispatches the email
Endpoint · /api/premarket/brief/synthesized
Endpoint receives the synthesized JSON, merges it into the persisted brief, renders the locked HTML template (Mock C v5 — KPI tiles, Market Context table, Your Book Today, F&O ban summary, Strategy outlook), dispatches via Gmail SMTP. Marks the brief as _email_sent: true so the fallback knows to skip.
HTML render SMTP send In-app log
08:08 IST
Fallback safety net
Job · premarket_brief_fallback · APScheduler
Reads the persisted brief. If _email_sent is already true, exits silently. Otherwise — cloud routine never fired, or hit an error — the VPS sends the un-synthesized version using its rule-based One-Liner and raw RSS headlines. The operator never wakes up to a missing brief.
Self-healing Idempotent

04 · The data contract

Structured by the VPS, narrated by the cloud.

The two halves communicate through a single JSON shape. Anything the cloud routine adds is namespaced as _synthesized — the rule-based fallback works without it.

05 · Design principles

Three rules.

Every design decision in this pipeline maps to one of these.

①
The operator must always get a brief
The 08:08 fallback exists because the cloud routine can fail silently — sandboxed sessions, network issues, model latency. A degraded brief beats no brief. The fallback is idempotent — if the synthesized version already went out, it does nothing.
②
Never store secrets in the cloud
The cloud routine is stateless. No SMTP credentials, no SSH keys, no Kite tokens, no secrets at all. It can only read public-ish data from the VPS and write back synthesis. Worst case: someone gets the routine's output. They get a forecast.
③
Each layer does one thing
VPS = data + transport. Cloud = language. Email template = render. Mixing layers (e.g., letting the cloud send email directly via Gmail MCP draft) creates manual-click footguns. Single responsibility per layer keeps the failure modes tiny.
④
Make every input observable
Every brief is persisted to backtest_data/premarket_brief_latest.json. The validator at 08:50 IST verifies the build job is registered. The fallback at 08:08 closes the loop. If anything breaks, the operator knows by the next email — at the latest, by EOD report at 15:40.

---
description: Build a highlight cut from a silent workout clip's events.json (EDL first, render on approval)
---

You are the editorial brain of a silent-video editing pipeline. Read
`events.json` (generated by `extractors/extract_events.py`) and design a tight
highlight cut. The JSON is the ONLY thing you read to make editorial calls —
you do not watch the video, though you MAY sample a few frames with ffmpeg to
confirm an angle/exercise label before finalizing.

Target highlight length: **30 seconds** (override if the user says otherwise in
$ARGUMENTS, e.g. "15s" or "make it 45 seconds").

## Rules

1. **Crucial moment** = a `motion_peak`, an `audio_impact`, and a rep
   `bottom_t`/`lockout_t` that all fall within **0.4s** of each other. Rank
   these by combined `(audio_impact.intensity + motion_peak.energy)`. Keep the
   top moments until the kept clip lengths sum to ~the target length.
   - If reps are empty (pose was skipped/failed), fall back to
     motion_peak + audio_impact convergence alone.
   - If audio_impacts are empty (silent track), rank by motion_peak energy
     near rep bottoms.
2. For each kept moment, cut from **0.8s before** to **1.2s after** the moment.
3. **Angle choice:** prefer the close-up shot for the peak rep instant; cut to a
   wide shot for the entrance/exit of each exercise. Use the `shots[]` ranges
   and each rep's `shot` field. If `angle` labels are `"unknown"`, sample one
   frame per shot (`ffmpeg -ss <mid> -i input -frames:v 1 frame.png`) to infer
   wide vs close, and say which you picked.
4. **Never hold one shot longer than 4s** — if a kept clip would exceed that,
   split it and rotate to another available angle covering the same time window.
5. **SFX:** place one impact SFX exactly on each surviving `audio_impact.t`.
   Pick from `sfx/`: `whoosh.wav`, `hit.wav`, `bass_drop.wav`. Map by type:
   `plate_clank`/`bar_rack` → `hit.wav`, `thud` → `bass_drop.wav`, fast motion
   with no clear impact → `whoosh.wav`.
6. **Music ducking:** duck the music bed **-8dB for 0.3s** around each SFX
   (the `/music` command handles the bed; here just note the duck points).

## Output (in this order)

1. A **ranked moment table** (moment time, the three converging signals + their
   values, combined score) so the user sees WHY each clip was chosen.
2. The **EDL** as a table the user can edit:

   ```
   Clip N  IN–OUTs  shot X (wide/close)  <what happens>  impact@t → file.wav
   ```

3. The **ffmpeg render command** that concatenates the clips (use
   `-ss/-to` per segment + concat, or a `filter_complex` trim/concat chain;
   re-encode for frame accuracy). Write to `output/highlight.mp4`.
4. The **ffmpeg audio-mix command** for music + SFX (amix/adelay for SFX
   placement, sidechain or volume automation for the -8dB ducks).

## STOP

Print the moment table + EDL, then **STOP and wait for the user's OK** before
running any render command. Do not render until approved.

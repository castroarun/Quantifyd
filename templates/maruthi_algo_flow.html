Execution
Trigger Fill Detection & Post-Fill Actions
Fill Detection
Paper Mode:
Tick-level simulation — checks every tick
BUY fills when LTP ≥ trigger price
SELL fills when LTP ≤ trigger price

Live Mode:
KiteTicker on_order_update callback
Zerodha pushes order status via WebSocket
COMPLETE → activates position instantly
CANCELLED/REJECTED → removes position
Fill price = average_price from Kite
No polling needed. Zerodha pushes order updates through the same WebSocket we use for tick data. Fill detection is instant — not 30-second or candle-close delayed.
Post-Fill Actions
Triggered immediately on fill detection:

1. Activate position (PENDING → ACTIVE)
2. Place protective option:
  • BULL → Buy protective PUT (5% OTM)
  • BEAR → Buy protective CALL (5% OTM)
3. Recalculate hard SL from master ST + ATR
4. Dashboard updates automatically via state API
Cross-Day Order Persistence
Problem: Zerodha cancels all unfilled SL-L orders at 3:30 PM EOD
Solution: Scheduled job at 9:16 AM re-places all PENDING positions on Kite
Contract Roll: If futures expiry changed overnight, updates tradingsymbol before re-placing
Gap Protection
Problem: Market opens past the trigger price (gap through) — SL-L fills at a terrible price that may retrace

Detection (9:16 AM):
• SELL trigger: LTP already below trigger → gap down past level
• BUY trigger: LTP already above trigger → gap up past level
• Order is NOT re-placed on Kite. Position marked GAP_PENDING

Confirmation (9:21 AM):
After 5 minutes of market confirmation:
• Gap filled? Price retraced back past trigger → Cancel (was a fake move)
• Signal reversed? Regime changed → Cancel
• Gap holds + signal intact? Enter at MARKET + immediate full hedges:
   1. Protective option (5% OTM insurance)
   2. Short option (1 strike OTM premium income)
   3. Hard SL recalculated from master ST + ATR
Signal Reversal Handling
Before placing new trigger:
• Cancel any opposite-direction pending orders (DB + Kite)
• Skip if same-direction trigger already pending
On Kite: kite.cancel_order() called for live orders before new placement

Safety
Trailing Hard Stop Loss
Formula:
buffer = 1.0 × ATR(7) on 30-min

BULL: SL = Master ST − buffer
  → Only moves UP (tightens toward price)

BEAR: SL = Master ST + buffer
  → Only moves DOWN (tightens toward price)
Tick-Level Check:
Runs on every incoming tick
Does NOT wait for candle close

On breach:
Close ALL positions immediately
Set regime = FLAT
Wait for fresh MASTER signal to re-enter
Hard SL resets fresh on every regime change (no carry-over from previous regime). Within a regime, it only moves in the favorable direction — never gives back ground.

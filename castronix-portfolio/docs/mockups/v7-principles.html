Design Principles Showcase

Five creative approaches to present engineering principles. Exploring how to make abstract concepts tangible and memorable.

1. Contrast Cards 2. Visual Equations 3. Real Examples 4. Flip + Contrast (Combined) 5. One Metric
01 Approach

Instead of / We do

Show the anti-pattern alongside the principle in action. Immediate clarity through contrast.

DRY Don't Repeat Yourself
Instead of

Copy-paste this validation logic to all 12 forms

We do

One useValidation hook, 12 forms, 0 duplicates

KISS Keep It Simple
Instead of

Build a state machine with 47 transitions for workout tracking

We do

Simple array of sets with timestamps. Done.

YAGNI You Aren't Gonna Need It
Instead of

Add multi-tenancy support "in case we need it later"

We do

Ship single-tenant MVP. Iterate when needed.

TDD Test-Driven Development
Instead of

Write tests after shipping, "when we have time"

We do

Test first. Red, green, refactor. Always.

03 Approach

Real Project Examples

Ground each principle in actual decisions from the portfolio. Show, don't tell.

R
KISS
REPPIT

We could have built a complex state machine for workout tracking. Instead, we used a simple array of sets with timestamps. 47 lines vs 3.

Zero bugs in workout logging since launch
N
DRY
NoteApp

Authentication logic was scattered across 8 components. One useAuth hook later: consistent behavior, single update point, 60% less code.

Auth changes now take minutes, not hours
P
YAGNI
PRIMMO

Multi-property comparison was on the roadmap. We shipped single-property analysis first. Users loved it. Multi-property is now V2.

Shipped 3 weeks earlier with validated core
